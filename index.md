## What I Build

Three things every project gets:

**1. Test Strategy** — risk-based, layered, verifiable coverage numbers. Smoke → E2E → API Contract → Mutation → DORA Metrics. No test without a traced risk.

**2. CI/CD Quality Gates** — typecheck, lint, audit, smoke, contract, mutation. One gate per failure type. No manual approvals for automation decisions.

**3. Metrics That Ship** — DORA, CFR, MTTR in Grafana dashboards, not static documents. Real-time coverage monitoring linked to test execution.
